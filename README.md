# ДИКО

Интернет-магазин товаров для необычных питомцев: рептилий, ежей, птиц, грызунов, нутрий и насекомых.

Проект сделан как учебное frontend-приложение на React и TypeScript. Демонстрирует основные сценарии интернет-магазина: просмотр каталога, поиск, фильтрацию, карточку товара, корзину, оформление заказа, личный кабинет, бонусы, промокоды и мини-игру.

## Стек

- React 18 - построение пользовательского интерфейса из компонентов.
- TypeScript - типизация данных и логики приложения.
- Vite - быстрый запуск проекта и production-сборка.
- CSS - стили, адаптивная верстка, светлая и темная темы.
- lucide-react - иконки интерфейса.
- localStorage - хранение демонстрационных данных в браузере.

## Возможности

- Главная витрина с промо-слайдером, категориями, брендами и подборками товаров.
- Каталог из 24 товаров, 9 категорий и 6 типов питомцев.
- Поиск по товарам с учетом синонимов.
- Фильтры по питомцу, категории, бренду, назначению, цене, весу, возрасту, рейтингу и наличию.
- Сортировка по популярности, цене, рейтингу, новизне и размеру скидки.
- Переключение вида каталога: сетка или список.
- Карточка товара с изображением, характеристиками, описанием, отзывами и похожими товарами.
- Избранное, сравнение товаров и недавно просмотренные товары.
- Корзина с изменением количества, удалением товаров и расчетом итоговой суммы.
- Промокоды и ограничение скидки размером заказа.
- Оформление заказа с валидацией полей.
- Авторизация, регистрация и выход из аккаунта.
- Личный кабинет с профилем, заказами, избранным, бонусами, промокодами и отзывами.
- Бонусная программа с уровнями лояльности.
- Советы по уходу за питомцами на основе состава корзины.
- Мини-игра для получения бонусов и промокодов.
- Светлая и темная тема.
- Версия интерфейса для слабовидящих.
- Всплывающие уведомления о действиях пользователя.

## Основные сценарии

1. Пользователь открывает главную страницу и переходит в каталог.
2. В каталоге он ищет товар, применяет фильтры и сортировку.
3. Открывает карточку товара, смотрит характеристики и отзывы.
4. Добавляет товар в корзину, избранное или сравнение.
5. В корзине применяет промокод и оформляет заказ.
6. После заказа получает бонусы.
7. В личном кабинете смотрит заказы, бонусы, промокоды и отзывы.
8. В мини-игре может получить дополнительные бонусы или промокод.

## Особенности реализации

- Проект работает без backend-сервера и базы данных.
- Корзина, пользователь, заказы, отзывы, избранное, сравнение, тема и бонусы сохраняются в `localStorage`.
- Данные каталога отделены от интерфейса, поэтому товары можно расширять отдельно от компонентов.
- Для каталога используются мемоизированные вычисления, чтобы фильтрация и сортировка не пересчитывались без необходимости.
- TypeScript-типы описывают ключевые сущности проекта и снижают риск ошибок при изменении данных.
- В корзине есть логика подсказок по уходу: система анализирует выбранные товары и показывает полезные рекомендации.

## Ограничения

- Оплата, доставка и регистрация реализованы как демонстрационные сценарии.
- Данные хранятся только в текущем браузере пользователя.
- После очистки localStorage демонстрационные заказы, избранное и пользовательские данные будут сброшены.
